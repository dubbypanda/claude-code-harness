# README Visual Asset Manifest

Date: 2026-05-24
Status: approval pending

This manifest defines the README visual refresh, but does not claim that final
images have been generated. Image generation and README image integration are
blocked until the user approves one visual direction.

## Model / Tool Boundary

- Requested label: `GPT-image2.0`
- Official OpenAI API docs checked on 2026-05-24:
  - https://platform.openai.com/docs/guides/image-generation
  - https://platform.openai.com/docs/guides/tools-image-generation/
  - https://platform.openai.com/docs/models/gpt-image-1.5
- Observed official API model family: `gpt-image-1.5`, `gpt-image-1`,
  `gpt-image-1-mini`; the Responses API image tool uses GPT Image models
  through `image_generation`.
- Decision: `GPT-image2.0` remains an unverified label in this repo. For API
  generation, use `gpt-image-1.5` unless the user explicitly provides a
  different local tool or ChatGPT UI route. Do not write "generated by
  GPT-image2.0" into README or CHANGELOG until that path is verified.

## Shared Constraints

- White background.
- Use the existing `docs/images/claude-harness-logo-with-text.png` tone as
  brand reference.
- Do not use unsupported host logos.
- Do not imply Cursor, GitHub Copilot CLI, Codex app, or Antigravity CLI are
  supported.
- Text-bearing images must have separate English and Japanese outputs.
- Avoid internal code names and release-history metaphors.
- Keep text sparse enough to survive image generation.
